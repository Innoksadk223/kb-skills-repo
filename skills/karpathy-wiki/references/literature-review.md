# Literature Review Matrix

Create a literature-review matrix when sources must become thesis-ready writing material.

The matrix is a route map for reading and writing. It should help answer: which literature exists, what each source contributes, what method it uses, what it cannot prove, which claim it supports or challenges, and where it fits in the paper.

## Where It Lives

Default location:

```text
synthesis/文献综述矩阵-主题.md
```

Use `synthesis/` because the matrix is a writing entry page. It should link to durable `claims/`, `concepts/`, `entities/`, `comparisons/`, or `debates/`.

Do not turn the matrix into a permanent evidence bank. If a row contains a reusable proposition, create or update a claim.

## When To Create One

Use this for:

- 5+ papers or sources on one research problem;
- writing a literature review, opening report, proposal, or thesis chapter;
- comparing methods, samples, concepts, or schools;
- finding research gaps or missing counterarguments;
- preparing "who says what" summaries.

## Row Fields

Use this compact table first:

| Source | Object | Method | Core view | Contribution | Limit | Wiki use |
|---|---|---|---|---|---|---|

Expand only when needed:

| Field | Meaning |
|---|---|
| Source | author/year/title or wiki entity page |
| Object | population, text, case, institution, theory, or data |
| Method | conceptual, qualitative, quantitative, mixed, review, archival |
| Core view | one-sentence thesis |
| Contribution | what this source adds to the project |
| Limit | sample, method, scope, inference, translation, or theory boundary |
| Wiki use | claim, concept, debate, chapter, or paragraph role |

## Workflow

1. Read the raw sources or existing wiki pages.
2. Group sources by research question, method, theory, or chronology.
3. Fill the matrix with short, source-grounded rows.
4. Create/update claims for reusable propositions.
5. Create/update debate pages for durable disagreements.
6. Add gaps: missing method, missing author, missing counterevidence, missing concept page.
7. Update `index.md` and `log.md`.

## Output Template

Use `references/templates/literature-review-template.md`.

## Quality Rules

- Each row should have a writing purpose, not just a summary.
- Do not mix "source says" and "I infer" in the same cell.
- Mark method limits explicitly.
- Link to wiki pages where possible; raw paths remain plain text.
- Keep the matrix short enough to scan. Split by theme if it grows too wide.
